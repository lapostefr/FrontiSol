Méthodes d'utilisation

Il y a plusieurs manière d'accéder et pouvoir utiliser Solaris


  1. Si vous faites partie de l'organisation La Poste, l'utilisation et les mises à jours , via la 'librairie partagée' se fait de manière quasi automatique via le panneau d'assets. A chaque mise à jour, une notification apparait vous permettant d'upgrader les composants présents dans votre fichier.
  2. Si vous faites partie d'une autre organisation ou avez un compte professionnel : il est nécessaire de récupérer les fichiers (.fig) afin de les publier dans votre environnement pour utilisation en 'librairie partagée' dans vos fichiers de travail. A noter que les montées de version doivent être faites manuellement (procédure de publication / mise à jour décrites ci-dessous).
  3. Si vous avez un compte gratuit, vous n'avez la possibilité de publier les styles uniquement. Pour utiliser les composants vous devrez travailler directement dans le fichier "Composants".

Publier Solaris dans son organisation

Afin de pouvoir les utiliser en librairie partagée disponible dans le panneau Asset, importez les .fig dans un projet d'équipe, puis publiez les styles (couleurs, typo, etc) et composant dans votre Organisation (Asset > Team Librairies > Publish current file)


  1. Déplacez les fichiers dupliqués dans un projet d'équipe dédié.
  2. Une fois les fichiers rangés dans le projet et l'équipe de votre choix, vous pouvez publier les fichiers en cliquant sur le nom du fichier puis “Publish styles and components” :
  • Commencez par publier le fichier des Fondamentaux : Colors, Typographie, Grids, Icons, Logo.
  • Une fois ce dernier publié, il vous faudra restaurer les liens dans le fichier “Component” . Pour ce faire, ouvrez la modale “Team library”, activez l’utilisation des fichiers “Fondamentaux” récemment publiés, puis cliquez sur “Missing libraries”. Choisissez le fichier récemment publié comme librairie de remplacement, puis sur “Swap libraries”.
  • Ceci fait, vous pouvez publier le fichier “Composants” dans votre projet.
  • Vous pouvez désormais utiliser ces fichiers comme librairie partagée dans vos fichiers via le panneau d'asset.


Si vous n’avez qu’un compte gratuit, vous ne pourrez pas publier les composants, mais seulement les styles (de couleur, de typographie). Il vous faudra donc travailler directement dans le fichier “Component”, en créant une nouvelle page dans l'arborescence.

Mettre à jour Solaris suite à une montée de version

﻿Voir la documentation de Figma﻿

Afin de mettre à jour la librairie publiée dans votre organisation suite à une montée de version :

  • supprimez la version précédente,
  • importez la nouvelle version,
  • publiez la nouvelle version
  • restaurez les liens dans vos fichiers locaux en utilisant la fonctionnalité “Swap libraries”.


Chaque nouvelle version de Solaris est annoncée dans le canal teams Solaris Design System General


Au sein de l'organisation La Poste , la mise à jour est 'automatique' lors de la publication, et une fenêtre de notification 'Component update' apparait dans l'interface Figma, vous permettant d'appliquer (et de comparer) les modifications sur vos fichiers.


En dehors de l'organisation La Poste, la mise à jour est manuelle. Il faut donc récupérer les nouveaux fichiers sur l'espace Team ou le fichier zip (plus haut dans cette page).


Pour mettre à jour vos librairies locales suite à une montée de version, il vous faudra répéter la manœuvre d'installation, et utiliser “Swap libraries” pour maintenir les liens dans vos projets :


  • Une fois ceci fait, vous pouvez supprimer les anciennes libraires 1.2.0 de votre espace local. Tous les liens précédemment effectués apparaîtront comme “Missing libraries”
  • Publiez la version 1.3.0 en suivant la méthode présentée dans le paragraphe précédent, sans oublier de restaurer les liens entre Fondamentaux et Composants.
  • Ensuite, vous pouvez restaurer les liens dans vos fichiers locaux en utilisant la fonctionnalité “Swap libraries”.
  • Si vous avez des libraires ou des composants intermédiaires, veillez à les mettre également à jour, en respectant autant que possible l’ordre de publication et restauration des liens suivants :
  • Fondamentaux > Composants > Librairies intermédiaires éventuelles > Fichiers de design.d

Quelques infos sur l'utilisation de la librairie

Attention

Si vous détachez des symboles de la librairie pour les modifier, vous ne bénéficierez plus des mises à jour automatiques pour ces éléments.

La grille

La grille permet de structurer le contenu d’une page. Pour en comprendre son fonctionnement consultez sa page dédiée (dans la doc et dans le fichier Grid). Dans Figma, les grilles sont disponibles en styles de composition par défaut. Pour les utiliser, rendez-vous dans la section “Layout grid” du panneau de propriétés d’une Frame.


Les "design tokens"

Les tokens sont de plus en plus utilisés au sein des composants, et une documentation d'utilisation pour vos besoins 'custom' (nouveaux composants ou customisation de mises en page) sera bientôt disponible dans une section dédiée de ce site.


Les composants

Les composants sont disponibles sous la forme de symboles dynamiques dans la librairie partagée.

Solaris étant en constante évolution, les composants actuels peuvent être amenés à être modifiés, et d’autres composants peuvent être ajoutés.
